Privacy Policy for Salesforce QuickField

Effective Date: May 16, 2025

Salesforce QuickField is a Chrome extension that helps Salesforce administrators quickly create custom fields and assign field-level security (FLS) to profiles. We are committed to protecting your privacy and maintaining transparency about how data is handled.

1. Information We Collect

We do not collect, store, or sell personally identifiable information (PII) or sensitive user data.

To perform its core functionality, the extension sends field creation metadata (e.g., field labels, API names, and permissions) to a secure backend service hosted on Render. This communication is used solely to process field creation and return metadata configurations. No session IDs, passwords, or PII are stored or shared.

2. Permissions Used

  • activeTab: To access the current Salesforce tab for context-aware operations.
  • scripting: To inject scripts into the Salesforce UI to read metadata.
  • clipboardWrite: To allow users to copy their session ID when they choose to share temporary access.
  • storage: To remember user preferences and field configuration settings locally.
  • Host permissions: (https://*.salesforce.com/) To operate only within the Salesforce domain.

3. Data Transmission to Backend

The extension communicates with a Node.js backend hosted on Render to generate Salesforce field metadata. Only non-personal configuration data is transmitted. This data is not stored or shared with any third parties.

4. Third-party Access

Salesforce QuickField does not use third-party analytics, tracking, or advertising. The only external service accessed is the backend API used for field generation.

5. Changes

Any future changes to this policy will be posted here. Continued use of the extension indicates acceptance of any updates.
